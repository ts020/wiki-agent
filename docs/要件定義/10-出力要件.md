# 10. 出力要件

## 構造（v0.4）

機械生成物と原本取り込みをトップレベルで区別するため、フラットな命名で分離する。

```
repo-wiki/
├── index.md              # トップ目次（ここから読む）
├── overview/             # 機械生成: 技術スタック・エントリポイント・テスト構造
├── development/          # 機械生成: ビルド/テストコマンド案内
├── code-nodes/           # 機械生成: コード由来ノード（ディレクトリ単位）
│   └── <path>/_symbols.md    # シンボル 100 件超過時の退避先
├── note-index/           # 機械生成: 手書き md の索引ページ
│                             #   （Summary / Structure / Related / Backlinks
│                             #    + 原本へのリンク。本文コピーは含めない）
├── imported/             # 原本取り込み: 手書き md の本文コピー
│                             #   （wikilink を Markdown リンクに変換するのみ）
├── tags/                 # 機械生成: タグ索引（ネストタグは tags/<a>/<b>.md）
└── _unresolved.md        # 機械生成: 未解決 wikilink 一覧
```

## 生成区分

| 区分 | ディレクトリ | 内容 |
|------|-------------|------|
| 機械生成 | `overview/`, `development/`, `code-nodes/`, `note-index/`, `tags/`, `index.md`, `_unresolved.md` | 走査・解析結果から組み立てる。`repo-wiki/` を丸ごと削除しても失われるのは生成物のみ |
| 原本取り込み | `imported/` | ユーザが書いた md の本文コピー。`[[wikilink]]` のみ Markdown リンクに変換し、それ以外は無改変 |

## 出力パス規則

- コード由来ノード: `code-nodes/<対象からの相対パス>.md`
- ノート索引: `note-index/<対象からの相対パス>`
- ノート原本コピー: `imported/<対象からの相対パス>`
- basename は元ファイル名を保持。衝突時は末尾に `-N` を付与
- パスに含められない文字（OS 制約）は `_` に置換

## リンクの向き先

- **本文中の wikilink (imported/ 配下のファイル内)**: 他ノートの `imported/<path>` を指す（読み進め体験を維持）
- **Related / Read next / Backlinks (note-index/ 配下)**: 他ノードの `note-index/<path>` または `code-nodes/<path>` を指す（索引を横断する体験）
- **note-index ページの「原本へ」リンク**: 対応する `imported/<path>` を指す
- **タグ索引ページのリンク**: 各ノードの索引ページ（note-index または code-nodes）を指す

## index.md
- 全体説明
- 技術スタック
- 各セクション（Overview / Code / Notes / Tags / Unresolved）への導線

## ノード制約
- 短い（目安 100 行以下）
- 自己完結
- 次の導線を持つ
