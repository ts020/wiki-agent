# 10. 出力要件

## 構造

```
repo-wiki/
├── index.md
├── overview/            # 技術スタック、エントリポイント、テスト構造
├── directories/         # コード由来ノード（ディレクトリ単位）
│   └── <path>/_symbols.md   # シンボル 100 件超過時の退避先
├── notes/               # ノート由来ノード（取り込んだ手書き md）
├── tags/                # タグ索引（ネストタグは tags/<a>/<b>.md）
├── development/         # セットアップ・ビルド・テスト手順（抽出可能な範囲で）
└── _unresolved.md       # 未解決 wikilink 一覧
```

## 出力パス規則
- コード由来ノード: `directories/<対象からの相対パス>.md`
- ノート由来ノード: `notes/<対象からの相対パス>` をベースに配置。`docs/foo.md` → `notes/docs/foo.md`
- ノード basename は元ファイル名を保持。衝突時は末尾に `-N` を付与
- パスに含められない文字（OS 制約）は `_` に置換

## index.md
- 全体説明
- 技術スタック
- ノード導線
- タグ索引入口
- 主要ノート入口

## ノード制約
- 短い（目安 100 行以下）
- 自己完結
- 次の導線を持つ
