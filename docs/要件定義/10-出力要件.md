# 10. 出力要件

## 構造（v1.0）

トップレベルにフラットに配置し、名前だけで役割が分かる形にする。

```
md-wiki/
├── index.md            # 入口。ノート数・索引への導線・未解決サマリ
├── notes/              # ノート本体（原本コピー + wikilink 変換 + 末尾に自動セクション）
│   └── <input からの相対パス>
├── tags/               # タグ索引（ネストタグは tags/<a>/<b>.md）
│   ├── index.md            # 全タグの入口（FR-07 により必須）
│   └── <tag>.md
├── headings/           # 見出し索引
│   └── index.md
├── links/              # リンク索引
│   └── index.md
└── _unresolved.md      # 未解決 wikilink 一覧（未解決 0 件でも空ページを生成）
```

### 出力先の扱い

`--out` で指定したディレクトリが既に存在する場合、実行前にその配下をクリーンアップしてから書き直す（FR-03）。差分マージは行わない。出力先が空でないディレクトリで、かつ内容が本ツール由来でない可能性があるケース（`index.md` と `notes/` のどちらも無い）では、誤削除を避けるためエラー終了する。

## 生成区分

すべて機械生成。`md-wiki/` を削除しても失われるのは生成物のみで、入力側の `.md` は無改変のまま残る。

## 出力パス規則

- ノート本体: `notes/<入力からの相対パス>`
- basename は元ファイル名を保持。出力先で衝突した場合は末尾に `-N` を付与
- パスに含められない文字（OS 制約）は `_` に置換
- タグ索引: `tags/<tag>.md`（ネストは `tags/<a>/<b>.md`）

## リンクの向き先

- **ノート本文中の wikilink**: 他ノートの `notes/<path>` を指す
- **`## Backlinks` / `## Related` の各リンク**: 他ノートの `notes/<path>` を指す
- **見出し索引からのリンク**: 該当ノートの見出しアンカー
- **タグ索引からのリンク**: 該当ノートのトップ
- **リンク索引からのリンク**: 該当ノートのトップ

## index.md

- プロジェクト名と生成日時
- サマリ（ノート数・タグ数・未解決数）
- 各セクションへの導線: `notes/`, `tags/`, `headings/`, `links/`
- 未解決が存在するときは `_unresolved.md` への警告リンクを表示

## ページ制約

- 索引ページは短く（目安 100 行以下）。超過時はサブページへの分割を検討する（v1 では主にタグでサブページ化される）
- ノート本体は行数制限を課さない（ユーザ成果物を改変しないため）
