# 10. 出力要件

## 構造（v1.1 / 断片化）

トップレベルはフラットに配置する。ノート本体は `fragments/` 配下に「入口ページ + 断片ページ群」として展開する。`<rel>` は **入力からの相対パスから `.md` 拡張子を除去したもの**（例: `docs/auth/session.md` → `docs/auth/session`）。

```
md-wiki/
├── .md-wiki/
│   ├── manifest.json                 # init/add の管理状態
│   └── catalog.json                  # context retrieval 用の内部索引
├── index.md                        # ルート概況（サマリ + 索引導線のみ。全ノート列挙は _index.md に委譲）
├── fragments/                      # ノート本体（入口 + 断片）
│   ├── _index.md                       # 階層サマリ: fragments 直下のノート／サブディレクトリ一覧（FR-16）
│   └── <rel>/
│       ├── _index.md                   # <rel> 配下が更にディレクトリを持つ場合の階層サマリ
│       ├── index.md                    # 入口ページ（h1 + プレフェイス + 断片一覧 + Backlinks + Related）
│       ├── <h2-slug>.md                 # 断片ページ（h2 単位。冒頭に Parent/Prev/Next ナビ）
│       └── <h2-slug>/                   # h3 再分割時のみ
│           ├── index.md                     # 殻ページ（h2 プレフェイス + 子断片一覧 + Parent ナビ + Backlinks）
│           └── <h3-slug>.md                 # 子断片ページ
├── tags/                           # タグ索引（ネストタグは tags/<a>/<b>.md）
│   ├── index.md
│   └── <tag>.md
├── headings/                       # 見出し索引
│   └── index.md
├── links/                          # リンク索引（ページ解像度）
│   └── index.md
├── agent/
│   ├── index.md
│   ├── pages/
│   │   ├── index.md
│   │   └── page-001.md
│   ├── terms/
│   │   ├── index.md
│   │   └── page-001.md
│   └── fields/
│       ├── index.md
│       └── <field>.md
└── _unresolved.md                  # 未解決 wikilink 一覧（未解決 0 件でも空ページを生成）
```

**命名規約**: 入口ページ `index.md` と階層サマリ `_index.md` を明確に分ける（Hugo 慣習）。`fragments/<rel>/index.md` はノート入口。`fragments/<rel>/_index.md` はそのディレクトリ配下のサマリ（`<rel>` が更にサブディレクトリを持つ場合のみ生成）。

例:

- 入力 `memo.md` → 入口 `fragments/memo/index.md`
- 入力 `docs/auth/session.md` → 入口 `fragments/docs/auth/session/index.md`
- 同 ノートに `## 背景`, `## 設計` の h2 がある → 断片 `fragments/docs/auth/session/背景.md`, `fragments/docs/auth/session/設計.md`
- `## 設計` が 300 行超 ＆ h3 が複数 → 殻 `fragments/docs/auth/session/設計/index.md` と子断片 `fragments/docs/auth/session/設計/<h3-slug>.md`

### 出力先の扱い

`init --out` で指定したディレクトリが既に存在する場合、実行前にその配下をクリーンアップしてから書き直す。出力先が空でないディレクトリで、かつ内容が本ツール由来でない可能性があるケース（`.md-wiki/manifest.json`、`index.md`、`fragments/` のいずれも無い）では、誤削除を避けるためエラー終了する。

`add --out` は `.md-wiki/manifest.json` を必須とする。`add` は出力先全体を削除せず、desired output と manifest の生成ファイル一覧を比較して、必要な生成ファイルだけを書き込み・削除する。manifest に載らない未管理ファイルは保持する。ただし生成予定パスに未管理ファイルが既に存在する場合は、上書きを避けるためエラー終了する。

`init` / `add` は出力先ごとに排他ロックを取得してから生成する。同じ出力先に対する別プロセスの実行中はエラー終了し、同時書き込みによる manifest や生成物の破損を防ぐ。ロックファイルは OS の一時ディレクトリ配下に作成し、正常終了・エラー終了のどちらでも削除する。

## 生成区分

`.md-wiki/manifest.json` と Markdown 出力は機械生成。`md-wiki/` を削除しても失われるのは生成物のみで、入力側の `.md` は無改変のまま残る。

## 出力パス規則

- 入口ページ: `fragments/<rel>/index.md`（`<rel>` は拡張子を除いた相対パス）
- 断片ページ（h2）: `fragments/<rel>/<h2-slug>.md`
- 子断片ページ（h3）: `fragments/<rel>/<h2-slug>/<h3-slug>.md`（h3 再分割時は h2 が殻ページ化する）
- 断片 slug は見出しスラグ仕様（FR-08）に従う。同一ノート内の slug 衝突・ファイル vs ディレクトリ衝突は FR-05 5.6 で解決
- 入力ディレクトリに `foo.md` と `foo/` が共存する場合も規則そのまま（`fragments/foo/index.md` と `fragments/foo/<child>/index.md` が同じ `fragments/foo/` 配下に入る）
- タグ索引: `tags/<tag>.md`（ネストは `tags/<a>/<b>.md`）
- パスに含められない文字（OS 制約）は `_` に置換

## リンクの向き先

- **本文中の `[[NoteName]]`**: 対象ノートの入口ページ（`fragments/<rel>/index.md`）
- **本文中の `[[NoteName#見出し]]`**: 対象 h2 の断片ページ。殻化時は殻 `index.md`、非断片化ノートなら入口 `index.md#<h2-slug>`
- **断片ページ冒頭ナビ (Parent / Prev / Next)**: 同一ノート内の入口／殻／兄弟断片
- **入口ページ・殻ページの `## Fragments`**: 直下の子断片ページ
- **`## Backlinks`**: 当該ページを参照している入口・殻・断片
- **`## Related`**: 入口ページのみに付与、対象ノートの入口ページへのリンク
- **見出し索引**: h1 は入口ページ、h2 は対応する断片ページ（殻化時は殻 index、非断片化時は入口 `#<slug>`）
- **タグ索引**: 該当ノートの入口ページ
- **リンク索引**: 断片／入口単位で「このページからの参照先」を列挙

## index.md

- プロジェクト名
- サマリ（ノート数・断片数・タグ数・未解決数）
  - 断片数は h2/h3 の断片ページ総数。入口・殻は数えない（FR-12）
- 各セクションへの導線: `fragments/_index.md`, `agent/index.md`, `tags/index.md`, `headings/index.md`, `links/index.md`, `_unresolved.md`
- ルート `index.md` は全ノートを直接列挙しない。全ノートへの到達は `fragments/_index.md` と `agent/pages/` に委譲する
- 未解決が存在するときは `_unresolved.md` への警告リンクを表示

## Context pack 出力

`md-wiki context` は生成 wiki ディレクトリへファイルを書き込まず、Markdown context pack を stdout に出力する。

context pack は以下を持つ:

- YAML frontmatter `md_wiki_context`
- context title
- schema context recipe に従う sections
- `Missing Required Evidence`（不足がある場合）
- `Source Trail`

context pack は `--budget` または context recipe の `default_budget_chars` を超えない。

## ページ制約

- **128k コンテキスト前提**: すべての生成ページは実装目安 **40,000 文字**以内を狙う（[§9](09-非機能要件.md) の非機能要件）。超過時は警告ログを出す
- 索引ページは短く（目安 100 行以下）。超過時はサブページへの分割を検討する（v1.2 ではタグと `_index.md` のみ分割対応、`headings/`・`links/` の階層分割は v1.3 送り）
- 断片ページは「AI が 1 度に読める小ページ」を狙う。h2 セクションが **300 行超（FR-05 5.3 の数え方）** かつ h3 が 2 個以上含まれる場合は h3 単位に再分割する
- 入口ページ・殻ページの本文長には上限を課さない（プレフェイス＋自動セクションが主な中身のため）。ただし 40,000 文字超は警告ログを出す
- `fragment: false` 指定時、および h2 を全く含まないノートでは、入口ページに全文をそのまま収める（行数上限を課さないが、40,000 文字超は警告）
