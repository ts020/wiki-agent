# 4. 用語定義

- **ノート**: 入力となる 1 個の `.md` ファイル
- **wiki ツリー**: 生成物のディレクトリ全体（入口ページ＋断片ページ＋索引群）
- **断片 (fragment)**: ノートを意味単位（既定では h2）で切り出した 1 ページ相当のかたまり。AI に部分読み込みさせる単位
- **入口ページ**: 1 ノートにつき 1 つ、ノート本来のプレフェイス（h1〜最初の h2 の手前まで）と断片一覧を持つページ。出力パスは `fragments/<rel>/index.md`
- **断片ページ**: ノート本文の各 h2（必要に応じ h3）に対応する独立ページ。出力パスは `fragments/<rel>/<slug>.md`
- **殻ページ**: h2 セクションを h3 単位に再分割した際の入口代替ページ。`fragments/<rel>/<h2-slug>/index.md`
- **索引**: タグ・見出し・リンクなど、ノート／断片を横断的に列挙するページ
- **タグ索引**: フロントマター `tags` を集計したページ群（`tags/<tag>.md`）。リンク先は入口ページ
- **見出し索引**: 全ノートの h1〜h2 を集約したページ。h1 は入口ページへ、h2 は対応する断片ページへ張る
- **リンク索引**: 断片ページ／入口ページ間の参照関係を一覧化したページ
- **wikilink**: Obsidian 由来の `[[NoteName]]` / `[[NoteName#見出し]]` 記法。出力時に標準 Markdown リンクへ変換する。`#見出し` 付きは該当 h2 の断片ページへ解決する
- **バックリンク**: 当該断片ページ（または入口ページ）を参照している他の断片／入口の一覧
- **フロントマター**: `.md` 先頭の `---` で囲まれた YAML ブロック
- **manifest**: `.md-wiki/manifest.json`。入力 root、再帰設定、source hash、生成ファイル hash を記録し、`add` の差分反映に使う管理ファイル
- **増築**: 入力ディレクトリに `.md` を追加・変更・削除し、`md-wiki add` で wiki を現在状態へ更新すること
