# 12. エラー処理

## 入力検証（実行を継続しない）
- `<INPUT>` が存在しない → エラー終了
- `<INPUT>` がファイルかつ拡張子が `.md` 以外 → エラー終了
- `<INPUT>` がディレクトリかつ直下・再帰共に `.md` が 1 件も見つからない → 警告ログを出し、空の wiki（index.md + 各索引の空ページ）を生成して正常終了

## 読み込み時（スキップして継続）
- 読めないファイル（権限等）→ 警告ログでスキップ
- UTF-8 として解釈できない → 警告ログでスキップ
- 1 MiB 超の UTF-8 `.md` → スキップせず `large_markdown` として large path へ分類
- 200 MiB 超の UTF-8 `.md` → 管理上限超過の理由を持つ `large_markdown` として扱い、公開前検証で個別確認
- NULL バイトを含む（バイナリ）→ 警告ログでスキップ
- シンボリックリンク → 辿らず無視

## パース時
- 不正な YAML フロントマター → 該当ブロックはフロントマター無しとみなし、本文全体をそのまま扱う（警告ログ）
- 不正な `[[wikilink]]` 記法 → 解釈を諦め原文を保持

## 生成時
- 出力先が書き込み不可 → エラー終了
- 出力先が存在し、中身が本ツール由来と推定できない（`index.md` と `fragments/` のどちらも存在しない、かつ空でもない）→ エラー終了し、誤って他人のディレクトリを破壊することを防ぐ
- 個別ノートの生成失敗 → 当該ノートはスキップしつつ、他ノート・索引は生成を続行。失敗したノートは警告ログで報告
- 索引の生成に必要な情報が欠落 → 空の見出しを持つ索引として出力する（壊れた wiki を残さない）
