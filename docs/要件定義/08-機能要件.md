# 8. 機能要件

## FR-01 CLI エントリ
`md-wiki <INPUT> [-r] [-o <DIR>]` として動作する。入力指定と出力指定の詳細は [§7](07-入力.md) に従う。

## FR-02 走査
- 入力がファイルなら当該ファイル 1 枚のみを対象とする
- 入力がディレクトリかつ `--recursive` なしなら、直下の `.md` のみ
- 入力がディレクトリかつ `--recursive` ありなら、配下を再帰走査

[§7.4 除外対象](07-入力.md#74-除外対象) と [§7.5 走査ガード](07-入力.md#75-走査ガード) に従う。

## FR-03 フル生成
毎回すべてのノート・索引を再生成する。差分更新は v1 スコープ外（[§14](14-将来拡張.md)）。出力先ディレクトリに既存の生成物がある場合は、実行前にクリーンアップしてから書き直す（マージはしない）。

## FR-04 ノート取り込み
[§7.2](07-入力.md#72-取り込み判定) のルールで対象 md を選別する。

## FR-05 ノート出力
取り込んだ `.md` は出力先の `notes/` 配下に、入力ルートからの相対パスを維持して配置する。

- 本文は原本を維持し、`[[wikilink]]` を標準 Markdown リンクへ変換する（FR-06）
- 先頭にはフロントマター由来のタイトル行を挿入しない（原本が持つ見出しを尊重する）
- 末尾に以下のセクションを自動で付与する（いずれも該当項目が 0 件ならセクション自体を省略する）：
  - `## Backlinks`: 当該ノートを参照している他ノート一覧（FR-10）
  - `## Related`: フロントマター `related` + 同タグ上位からの推定（FR-11）

自動セクションが 1 つでも付く場合、原本本文とは水平線（`---`）で区切る。

## FR-06 wikilink 解釈
Obsidian 互換の `[[...]]` 記法を解釈し、出力では相対パスの Markdown リンクへ変換する。

### 対応構文

| 構文 | 意味 | 出力 |
|---|---|---|
| `[[NoteName]]` | ノート参照 | `[NoteName](相対パス)` |
| `[[NoteName\|表示名]]` | エイリアス付き | `[表示名](相対パス)` |
| `[[NoteName#見出し]]` | 見出しアンカー | `[NoteName#見出し](相対パス#slug)` |
| `![[NoteName]]` | 埋め込み | v1 未対応。プレーンリンクに縮退 |

### 解釈対象外の領域

以下に含まれる `[[...]]` は解釈せず原文保持：

- フェンスドコードブロック内（``` または ~~~ で囲まれた領域）
- インラインコード内（`` ` `` で囲まれた範囲）
- HTML コメント内（`<!-- ... -->`）

数式ブロック・カスタム拡張等は v1 では特別扱いしない（将来拡張）。

### 解決規則

ターゲット名の候補を以下の優先順で選ぶ。上位で 1 件に確定しない場合のみ下位のルールに進む：

1. `aliases` / `title` に完全一致するノート
2. 出力先 basename（拡張子除く）に完全一致するノート
3. （ここまでで複数残ったとき）参照元と同一ディレクトリ配下のもの
4. （それでも複数残ったとき）最短パスのもの
5. （それでも複数残ったとき）相対パス文字列のアルファベット順

すべて空集合なら **未解決** とみなす。未解決時の本文表記は FR-13 を参照。

## FR-07 タグ索引
- フロントマターの `tags: [a, b]` を第一級タグとして扱う
- ネストタグ `auth/session` は `tags/auth.md` と `tags/auth/session.md` の双方に集計
- 本文内の `#tag` 記法は v1 では解釈しない（コードスニペット等との誤検出回避）
- `tags/<tag>.md` は自動生成し、該当ノートをフラットに列挙
- `tags/index.md` を生成し、全タグを列挙した入口ページとする
- 親タグと同名の子タグがあり `tags/auth.md`（ファイル）と `tags/auth/`（ディレクトリ）が衝突する場合: ファイル側を `tags/auth.md` として残し、サブタグは `tags/auth/` 配下に置く（OS / ファイルシステム上共存可能）

## FR-08 見出し索引
全ノートの見出し（h1〜h2）を横断集約した索引を生成する。

- 出力先: `headings/index.md`
- 各エントリは「見出しテキスト / 所属ノート名 / 該当見出しへのアンカーリンク」を持つ
- h3 以上は v1 では対象外（[§14](14-将来拡張.md)）

### 見出しスラグ仕様（再現性のため固定する）

1. Unicode NFKC 正規化
2. ASCII 英大文字は小文字化
3. 空白（`\s+`）は単一の `-` に置換
4. 次の文字は除去: 句読点 `` .,:;!?`'"()[]{}<>@#$%^&*+=|\/~ `` と制御文字
5. 残った連続 `-` は 1 個に畳み、先頭末尾の `-` を除去
6. 日本語・CJK・記号等は NFKC 後にそのまま残す（GitHub 風挙動）
7. 同一ページ内で衝突したら、2 番目以降に `-1`, `-2`, ... を付与（出現順）

## FR-09 リンク索引
ノート間の wikilink 参照関係を一覧化した索引を生成する。

- 出力先: `links/index.md`
- ノートごとに「このノートが参照しているノート」を列挙
- 逆向き（被参照）は各ノート末尾の Backlinks に出すため、リンク索引では省略してよい
- 未解決リンクは含めず、`_unresolved.md` に集約する

## FR-10 バックリンク
各ノートに「このノートを参照している他ノート一覧」を自動付与する。

- 各ノートページ末尾の `## Backlinks` セクションに列挙
- 参照元ノートが存在しなくなった場合は列に出さない（フル生成のため残らない）
- 参照が 0 件のノートでは `## Backlinks` セクション自体を出さない（FR-05）
- 同じノートから複数回参照されている場合でも一度だけ列挙する

## FR-11 Related
- フロントマター `related` を最優先で列挙（記載順を保持）
- それに加えて、共通タグが 2 つ以上あるノートを最大 3 件まで推定列挙
  - スコア: 共通タグ数の降順
  - 同点時のタイブレーク: 出力相対パス（`notes/<rel>`）の辞書順昇順
- 自身へのリンクは除外
- `related` で既に列挙済みのノートは推定列挙でも重複させない
- 候補が 0 件なら `## Related` セクション自体を出さない（FR-05）

## FR-12 index.md
ルートの `index.md` は以下を含む：

- プロジェクト名（入力ルートの basename、または `--out` 既定時は `md-wiki`）
- ノート数・タグ数・未解決数のサマリ
- `notes/` `tags/` `headings/` `links/` 各セクションへの導線
- `_unresolved.md` が非空なら警告リンクとして表示

生成日時は含めない（冪等性を保ち、`git diff` でのノイズを避けるため）。

## FR-13 未解決リンク
解決できなかった wikilink は `_unresolved.md` にまとめる。

- 参照元ノート・未解決ターゲット名・エイリアス・見出しを列挙
- ノート本文では **wikilink 記法のまま残し、直後に ` (未解決)` を付与** する
  - 例: `[[Foo]] (未解決)`、`[[Foo#sec|表示]] (未解決)`
  - CommonMark パーサはこのまま素通しし、リンク化はされない
- `_unresolved.md` が空（未解決 0 件）のときも空ページを生成する（存在保証）

## FR-14 走査上限

用語:
- **走査ファイル数**: 走査で発見した `.md` の総数（スキップされた md・フロントマター `wiki: false` も含む）
- **取り込みノート数**: 実際に `notes/` に出力した `.md` の数

上限:
- 走査ファイル数が 10,000 件を超えたら警告ログ（処理は継続）
- 取り込みノート数が 5,000 件を超えたら警告ログ（処理は継続）
- ディレクトリ深さが 20 段を超えたら警告ログ（処理は継続）
- 個別ファイルの上限は [§7.5](07-入力.md#75-走査ガード)

## FR-15 増築性
入力ディレクトリに `.md` を追加して再実行するだけで、以下がすべて自動更新される：

- `notes/` へのノート追加
- タグ索引の追記
- 見出し索引の追記
- リンク索引の追記
- バックリンク
- 未解決リンクの解決（追加したノートが既存未解決の対象だった場合）
