# 13. 受け入れ基準

v1 の完成条件。いずれも CI / 手動で実行可能なテストとして記述する。

## AC-01 CLI が起動する
`md-wiki --help` がゼロ終了し、使い方を stdout に出す。stdout には `init` と `add` が含まれる。

## AC-02 単一 md 入力で入口ページが生成される
`md-wiki init` に 1 枚の `.md` を渡すと、`./md-wiki/index.md`、`./md-wiki/.md-wiki/manifest.json`、`./md-wiki/fragments/<name>/index.md` が生成される。ノートが h2 を含む場合は `./md-wiki/fragments/<name>/<h2-slug>.md` が断片ページとして追加で生成される。

## AC-03 ディレクトリ入力（既定は再帰）
`.md` を含むディレクトリを渡すと、配下の `.md` がすべて `fragments/` に取り込まれ、入力相対パス（拡張子除く）が `fragments/` 以下にディレクトリ階層として維持される。入力を省略した場合はカレントディレクトリを同じ規則で走査する。

## AC-04 ディレクトリ入力（非再帰）
`--no-recursive` 付きでディレクトリを渡すと、直下の `.md` だけが `fragments/` に取り込まれる。サブディレクトリ配下の `.md` は無視される。

## AC-05 除外対象
`.git` `node_modules` `dist` `build` `target` と隠しディレクトリ（`.wiki` 以外）は走査されない。

## AC-06 wikilink 解決
`[[Foo]]` は対象ノートの入口ページ（`fragments/<rel>/index.md`）への相対リンクに変換される。`[[Foo#見出し]]` は対応する h2 の断片ページ（殻化時は殻 `index.md`、非断片化ノートなら入口 `index.md#<slug>`）へのリンクとなる。エイリアス付き・埋め込み記法（v1 は `[[...]]` と同じ扱いに縮退）のいずれも処理できる。

## AC-07 未解決リンク
存在しないターゲットへの `[[...]]` は本文末尾に ` (未解決)` として残り、かつ `_unresolved.md` に集約される。対象ノートは存在するが `#見出し` が存在しないケース、h1／h3 に一致した場合も未解決として扱う。

## AC-08 タグ索引
`tags: [a, b]` をフロントマターに持つノートは、`tags/a.md` と `tags/b.md` の双方に列挙される。ネストタグ `a/b` は `tags/a.md`（親側：子孫の `a/*` もすべて包含）と `tags/a/b.md` の両方に列挙される。リンク先は入口ページ。

## AC-09 見出し索引
`headings/index.md` に、取り込んだ全ノートの h1〜h2 が列挙される。h1 は入口ページへ、h2 は該当する断片ページへリンクする。非断片化ノートの h2 は入口ページ内のアンカーを指す。

## AC-10 リンク索引
`links/index.md` に、各断片ページ／入口ページが参照している他ページが列挙される。未解決参照は含まれない。参照元ページ見出しは `### <title>` 形式で、出力パス辞書順に並ぶ。

## AC-11 バックリンク
各ページ（入口・殻・断片）末尾の `## Backlinks` に、当該ページを参照している他ページ一覧が出る。参照が無いページではセクション自体を出さない。同じノート内の別断片からの参照も独立した参照として列挙される。並び順は参照元ページ出力相対パスの昇順。

## AC-12 index.md
ルートの `index.md` にノート数・断片数・タグ数・未解決数のサマリと、`fragments/_index.md`、`agent/index.md`、`tags/index.md`、`headings/index.md`、`links/index.md`、`_unresolved.md` への導線が含まれる。ルート `index.md` は全ノートや全入口ページを直接列挙しない。断片数は h2/h3 断片ページの合計で、入口ページ・殻ページは含まない。

## AC-13 増築と冪等性
- 既存の wiki がある状態で入力ディレクトリに `.md` を追加して `md-wiki add` を実行すると、index・タグ索引・見出し索引・リンク索引・バックリンク・未解決一覧・断片ページ・agent catalog がすべて追加分に合わせて更新される
- source が削除された場合、manifest に載る生成ページは `add` で削除される
- 未変更の生成ページは `add` で書き換えない
- manifest が無い出力先での `add`、入力 root 外の `add PATH`、未管理ファイルとの生成パス衝突はエラーになる
- 同じ出力先の `init` / `add` が既に実行中の場合は排他ロックによりエラーになり、成功後はロックが残らない
- **冪等性**: 追加後の入力セットに対して「出力先をまっさらな状態から `init` した結果」と「既存 wiki の上から `add` した結果」がバイト単位で一致すること

## AC-14 非破壊
実行後も入力側のファイル／ディレクトリの内容が一切変更されない。

## AC-15 オフライン動作
外部ネットワーク接続なし（`--offline` 的状態）で実行が完走する。

## AC-16 断片化とナビゲーション
複数の h2 を持つノートでは、h2 ごとに `fragments/<rel>/<h2-slug>.md` が生成される。各断片ページの冒頭は以下を満たす:

1. 1 行の blockquote ナビ `> Parent: ... (· Prev: ...)? (· Next: ...)?`
2. 直後に水平線 `---`
3. Parent は常に入口ページ（`index.md`）を指す
4. 先頭の h2 断片には Prev が含まれない
5. 末尾の h2 断片には Next が含まれない
6. 中間の h2 断片には Prev と Next の両方が含まれる

ナビの直後にその h2 見出しと本文が続く。

## AC-17 h3 再分割の閾値
1 つの h2 セクションの本文長（FR-05 5.3 定義）について、以下の境界動作を満たす:

| 本文長 | h3 の数 | 挙動 |
|---|---|---|
| ≤ 300 行 | 任意 | 通常断片 `<h2-slug>.md` |
| > 300 行 | 0〜1 個 | 通常断片（再分割しない） |
| > 300 行 | ≥ 2 個 | 殻化。`<h2-slug>/index.md` と `<h2-slug>/<h3-slug>.md` を生成 |

殻化した h2 は、入口ページの `## Fragments` セクションからも殻 `index.md` へリンクされる。

## AC-18 断片化オプトアウト
フロントマターに `fragment: false` を指定したノートは、入口ページ `fragments/<rel>/index.md` に全文をまとめるだけで、h2 単位の断片ページも h3 子断片ページも生成しない。`## Fragments` セクションも出さない。

## AC-19 単一 h2 ノート
h2 を 1 つだけ含むノートでも、入口ページ + 断片ページ 1 枚が生成される（入口にまとめずに分離する）。当該断片ページのナビは `> Parent: ...` のみ（Prev / Next なし）。

## AC-20 h2 無しノート
h2 を 1 つも含まないノート（h3 以下しか無い、もしくは見出し自体が無い）は、入口ページにのみ全文が収まり、断片ページは生成されない。

## AC-21 階層サマリ `_index.md`
`fragments/` 配下の各ディレクトリ（トップ `fragments/` を含む）に `_index.md` が生成され、直下のサブディレクトリと入口ページへのリンク、配下のノート数・断片数が載る。入口ページ (`index.md`) と同じディレクトリに共存しても衝突しない。

## AC-22 ルート index の縮減
ルート `index.md` には「全ノートの列挙」が含まれず、`fragments/_index.md` への導線と各索引へのリンクのみが並ぶ。

## AC-23 ページサイズ上限の警告
生成されたページが 40,000 文字を超えた場合、`warn` レベルのログにページパスと文字数が出る（処理は継続する）。

## AC-24 schema pack 読み込み
`--schema` に valid な schema pack YAML を指定すると `init` / `add` / `context` が schema id、version、fields、contexts を読み込める。不正 YAML、必須 field 欠落、未定義 field 参照はエラーになる。

## AC-25 catalog
`md-wiki init --schema <YAML>` は `.md-wiki/catalog.json` を生成する。catalog には schema id / version、generated page path、source path、source range、entities、tags、headings、extracted fields、outgoing links、backlinks が含まれる。

## AC-26 field catalog
schema pack が指定された場合、`agent/fields/index.md` と `agent/fields/<field>.md` が生成される。field catalog は該当 field を持つ generated page と source range を列挙する。

## AC-27 context pack 生成
`md-wiki context --wiki <DIR> --schema <YAML> --task <TASK>` は Markdown context pack を stdout に出す。出力は YAML frontmatter `md_wiki_context`、context title、schema の section order に従う本文、`Source Trail` を含む。

## AC-28 required field 欠落
context recipe の `required: true` field が見つからない場合、`context` は正常終了し、`Missing Required Evidence` に不足 field と対象条件を出す。

## AC-29 context budget
`context` の出力文字数は `--budget` または context recipe の `default_budget_chars` を超えない。budget 超過時も YAML frontmatter と `Source Trail` は保持される。

## AC-30 retrieval 決定性
同一入力、同一 schema、同一 `context` command では byte-identical な context pack が出力される。

## AC-31 no inference / no network
`context` は LLM、embedding、network、外部 API、schema pack 内の任意コード実行に依存しない。候補収集は catalog、明示 metadata、文字列一致、wikilink / backlink 1 hop に限定される。
