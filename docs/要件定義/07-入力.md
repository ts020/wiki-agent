# 7. 入力

## 7.1 CLI 形式

```
md-wiki <INPUT> [--recursive] [--out <DIR>]
```

| 引数 / オプション | 省略時 | 説明 |
|---|---|---|
| `<INPUT>` | 必須 | 単一の `.md` ファイルパス、または `.md` を含むディレクトリパス |
| `--recursive` / `-r` | off | ディレクトリ入力時に再帰的に走査する |
| `--out <DIR>` / `-o <DIR>` | `./md-wiki` | 出力先ディレクトリ |

### 入力種別ごとの挙動

| 入力 | `-r` | 読み込み対象 |
|---|---|---|
| ファイル（`.md`） | 無視される | そのファイル 1 枚 |
| ディレクトリ | off | 直下の `.md` のみ（再帰しない） |
| ディレクトリ | on  | 配下を再帰的に走査し、すべての `.md` を対象にする |

`.md` 以外の拡張子をファイル指定した場合はエラーにする。

## 7.2 取り込み判定

対象 `.md` について以下の順で評価する。先に該当したルールで確定。

1. フロントマターに `wiki: false` → **除外**
2. それ以外 → **取り込み**

`wiki: true` はフロントマター無しでも取り込まれるため冗長だが、明示目的で書くのは許容する。

**旧仕様との差分:** v0.x では `docs/` `notes/` `.wiki/` ディレクトリ規約や `README.md` 特例で判定していたが、md-wiki では入力が `.md` 群であることが前提のため「特段の理由がなければ取り込む」をデフォルトとする。

## 7.3 フロントマター仕様

フロントマターは YAML。先頭が `---\n` で始まり、独立行の `---` で閉じる。未知のフィールドは無視する。

| フィールド | 型 | 用途 |
|---|---|---|
| `wiki` | bool | 取り込み可否の明示指定（`false` で除外） |
| `title` | string | ノート表示名（未指定時は h1 → basename の順で決定） |
| `summary` | string | 索引・一覧表示で使用する短い説明文 |
| `tags` | string[] | タグ索引の集計対象。`a/b` のネスト表記可 |
| `related` | string[] | 追加の関連ノート（wikilink ターゲット、または出力相対パス） |
| `aliases` | string[] | wikilink 解決時の別名 |

例:

```yaml
---
wiki: true
title: 認証フローの設計メモ
summary: セッション維持方式の比較と採用案
tags: [auth, design]
related: [session-lifecycle, cookie-policy]
aliases: [認証メモ, auth-flow]
---
```

## 7.4 除外対象

ディレクトリ走査時、以下のディレクトリは走査しない（隠しディレクトリと VCS / ビルド生成物）：

- `.git`
- `node_modules`
- `dist`
- `build`
- `target`
- 名前が `.` で始まるディレクトリ（ただし `.wiki` は例外として走査する）

`.wiki` を例外にしているのは、ソースツリーに「ドキュメントは `.wiki/` 配下に隔離する」という運用慣習があり、旧 `repo-wiki` 時代にも採用されていた取り込み先のひとつだったためである。md-wiki では「取り込み判定はフロントマター `wiki: false` のみで除外」という方針に変わったが、ユーザが `.wiki/` を意図的に wiki 化したいケースを壊さないよう例外として維持する。

## 7.5 走査ガード

- ファイルサイズ 1 MiB 超はスキップ（警告ログ）
- NULL バイトを含むファイルはバイナリとみなしスキップ
- シンボリックリンクは辿らない（循環回避）
- UTF-8 として解釈できないファイルはスキップ（警告ログ）
