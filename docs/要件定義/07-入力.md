# 7. 入力

## 7.1 コード走査対象
対象ディレクトリ配下のソースコード全般。言語は問わない。
走査上のガード（巨大リポジトリ対応、FR-14 と共通）：
- ファイルサイズ 1 MiB 超はスキップ（警告ログ）
- NULL バイトを含むファイルはバイナリとみなしスキップ
- シンボリックリンクは辿らない（循環回避）
- 隠しディレクトリ（`.` 始まり）は原則スキップ。例外は `.wiki/` のみ

## 7.2 手書きノート取り込み
`.md` ファイルの取り込み判定は以下の優先順で行う（先に該当したルールで確定）：

1. フロントマターに `wiki: false` → **除外**
2. フロントマターに `wiki: true` → **取り込み**
3. プロジェクトルート直下の `README.md` → **取り込み**（index の overview 生成にも利用）
4. `docs/` / `notes/` / `.wiki/` 配下(再帰的) → **取り込み**
5. 上記いずれにも該当しない md → **無視**

この規約により、サブパッケージ配下の `README.md` 等を誤って大量取り込みすることを防ぐ。

### フロントマターで解釈するフィールド（未知フィールドは無視）

| フィールド | 型 | 用途 |
|------------|-----|------|
| `wiki` | bool | 取り込み可否の明示指定 |
| `title` | string | ノード表示名（未指定時は h1 または basename） |
| `summary` | string | Summary 欄に使用（未指定時は冒頭段落） |
| `tags` | string[] | タグ索引への集計対象。`a/b` のようなネスト表記可 |
| `related` | string[] | 追加の Related リンク（出力先パスまたはノート basename） |
| `aliases` | string[] | wikilink 解決時の別名 |

フロントマター例：
```yaml
---
wiki: true
title: 認証フローの設計メモ
tags: [auth, design]
related: [directories/auth.md]
aliases: [認証メモ, auth-flow]
---
```

## 7.3 除外対象（FR-05 と同一）
`.git`, `node_modules`, `dist`, `build`, `target` 配下は走査しない。

## 7.4 文字コード
入力ファイルは UTF-8 を前提とする。UTF-8 として解釈できないファイルはスキップ（警告ログ）。
