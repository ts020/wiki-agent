# 7. 入力

## 7.1 CLI 形式

```
md-wiki init [INPUT] [--no-recursive] [--schema <YAML>] [--out <DIR>]
md-wiki add [PATH] [--schema <YAML>] [--out <DIR>]
md-wiki context --wiki <DIR> --schema <YAML> --task <TASK> [--entity <TYPE:ID>]... [--time <EXPR>] [--query <TEXT>] [--budget <CHARS>]
```

| 引数 / オプション | 省略時 | 説明 |
|---|---|---|
| `init [INPUT]` | `.` | 単一の `.md` ファイルパス、または `.md` を含むディレクトリパスを初期生成する |
| `add [PATH]` | `init` の入力 root | 初期化済み wiki を入力 root の現在状態へ差分更新する。`PATH` は入力 root 配下のみ指定可 |
| `--no-recursive` | off | `init` のディレクトリ入力時に直下のみに限定して走査する。設定は manifest に保存され、`add` でも使われる |
| `--schema <YAML>` | なし | field 抽出と context recipe を定義する schema pack。指定時は `init` / `add` で field catalog と `.md-wiki/catalog.json` に schema 抽出結果を含める。`context` では必須 |
| `--out <DIR>` / `-o <DIR>` | `./md-wiki` | 出力先ディレクトリ |

旧形式 `md-wiki <INPUT> ...` は受け付けず、必ずサブコマンドを指定する。

### `context` 入力

`context` は生成済み wiki を対象に Markdown context pack を stdout に返す。

| 引数 / オプション | 必須 | 説明 |
|---|---:|---|
| `--wiki <DIR>` | yes | `.md-wiki/catalog.json` を持つ生成済み wiki |
| `--schema <YAML>` | yes | compile 時と同じ schema pack |
| `--task <TASK>` | yes | schema pack の `contexts` に定義された context recipe |
| `--entity <TYPE:ID>` | no | retrieval の明示起点。複数指定可 |
| `--time <EXPR>` | no | frontmatter または schema field の明示 metadata に対する一致条件 |
| `--query <TEXT>` | no | title、heading、tag、field text に対する文字列検索 |
| `--budget <CHARS>` | no | context pack の文字数上限。省略時は context recipe の `default_budget_chars` |

### 入力種別ごとの挙動

| 入力 | オプション | 読み込み対象 |
|---|---|---|
| ファイル（`.md`） | 無視される | そのファイル 1 枚 |
| ディレクトリ | 既定 | 配下を再帰的に走査し、すべての `.md` を対象にする |
| ディレクトリ | `--no-recursive` | 直下の `.md` のみ（再帰しない） |

`.md` 以外の拡張子をファイル指定した場合はエラーにする。

`init` は出力先に `.md-wiki/manifest.json` を生成し、入力 root、入力種別、再帰設定、source hash、生成ファイル hash を記録する。`add` はこの manifest が無い出力先ではエラーにする。

## 7.2 取り込み判定

対象 `.md` について以下の順で評価する。先に該当したルールで確定。

1. フロントマターに `wiki: false` → **除外**
2. それ以外 → **取り込み**

`wiki: true` はフロントマター無しでも取り込まれるため冗長だが、明示目的で書くのは許容する。

**旧仕様との差分:** v0.x では `docs/` `notes/` `.wiki/` ディレクトリ規約や `README.md` 特例で判定していたが、md-wiki では入力が `.md` 群であることが前提のため「特段の理由がなければ取り込む」をデフォルトとする。

## 7.3 フロントマター仕様

フロントマターは YAML。先頭が `---\n` で始まり、独立行の `---` で閉じる。未知のフィールドは無視する。

| フィールド | 型 | 用途 |
|---|---|---|
| `wiki` | bool | 取り込み可否の明示指定（`false` で除外） |
| `fragment` | bool | 断片化の可否。`false` で分割を行わず、入口ページに全文をまとめる（既定 `true`） |
| `title` | string | ノート表示名（未指定時は h1 → basename の順で決定） |
| `summary` | string | 索引・一覧表示で使用する短い説明文 |
| `tags` | string[] | タグ索引の集計対象。`a/b` のネスト表記可 |
| `related` | string[] | 追加の関連ノート（wikilink ターゲット、または出力相対パス） |
| `aliases` | string[] | wikilink 解決時の別名 |

例:

```yaml
---
wiki: true
title: 認証フローの設計メモ
summary: セッション維持方式の比較と採用案
tags: [auth, design]
related: [session-lifecycle, cookie-policy]
aliases: [認証メモ, auth-flow]
---
```

## 7.4 除外対象

ディレクトリ走査時、以下のディレクトリは走査しない（隠しディレクトリと VCS / ビルド生成物）：

- `.git`
- `node_modules`
- `dist`
- `build`
- `target`
- 名前が `.` で始まるディレクトリ（ただし `.wiki` は例外として走査する）

`.wiki` を例外にしているのは、ソースツリーに「ドキュメントは `.wiki/` 配下に隔離する」という運用慣習があり、旧 `repo-wiki` 時代にも採用されていた取り込み先のひとつだったためである。md-wiki では「取り込み判定はフロントマター `wiki: false` のみで除外」という方針に変わったが、ユーザが `.wiki/` を意図的に wiki 化したいケースを壊さないよう例外として維持する。

## 7.5 走査ガード

- 1 MiB 以下の UTF-8 `.md` は通常 Markdown として取り込む
- 1 MiB 超かつ 200 MiB 以下の UTF-8 `.md` はスキップせず、`large_markdown` として large path へ分類する
- 200 MiB 超の UTF-8 `.md` は管理上限超過の理由を持つ `large_markdown` として扱い、公開前検証では個別確認対象にする
- NULL バイトを含むファイルはバイナリとみなしスキップ
- シンボリックリンクは辿らない（循環回避）
- UTF-8 として解釈できないファイルはスキップ（警告ログ）

## 7.6 Schema pack YAML

schema pack は外部 YAML ファイルである。Rust 本体にドメイン固有の retrieval logic を直書きしない。

必須 top-level field:

| field | 型 | 説明 |
|---|---|---|
| `id` | string | schema pack ID |
| `version` | integer | schema pack schema version。v1 は `1` |
| `fields` | map | field 抽出定義 |
| `contexts` | map | context recipe 定義 |

任意 top-level field:

| field | 型 | 説明 |
|---|---|---|
| `description` | string | schema pack の説明 |
| `entity_types` | string[] | 使用できる entity type の一覧 |

v1 の field source は `frontmatter` と `heading` に対応する。v1 では scoring DSL、任意コード実行、外部コマンド実行を許可しない。
