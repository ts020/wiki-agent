# 1. 概要

## 1.1 名称
repo-wiki agent

## 1.2 一文定義
ローカルの指定ディレクトリ配下のコードベースおよび手書き Markdown ノートを解析し、AI および人間が少ないコンテキストで探索・理解できる Markdown ベースの wiki ツリーを生成する CLI エージェント。

## 1.3 背景
大規模または中規模のコードベースを理解する際、毎回リポジトリ全体を読み直すのは非効率である。特に AI を活用する場合、コンテキスト長の制約により、全文投入前提の理解方法はスケールしにくい。
一方で、AI (LLM) による要約生成は API 利用料がかさみ、再生成を繰り返すツールとしては運用コストが重い。
そのため、本システムは **AI を使わない静的解析・ルールベース処理** のみでコードベースを探索可能な木構造ドキュメントへ変換し、必要なノードのみ段階的に参照できる状態を作ることを目的とする。

## 1.4 目的
- コードベース理解の初期コストを下げる
- AI が必要最小限のノードのみを読んで対象箇所へ到達できるようにする
- 人間が index から段階的に構造を把握できるようにする
- 人間が書いた md ノートを投入するだけで、リンクとインデックスが自動更新される "増築型" wiki を実現する
- 後続の開発・分析タスクの前処理として機能する

## 1.5 生成物の性質
本システムの生成物は完全仕様書ではなく、**探索用の索引付き要約ツリー**である。
要約は AI による自然文生成ではなく、**構造抽出（見出しツリー・関数列挙・メタ情報）** による機械的な索引である。
