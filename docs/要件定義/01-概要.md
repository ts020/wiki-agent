# 1. 概要

## 1.1 名称
md-wiki

## 1.2 一文定義
ローカルの Markdown ファイルを入力に取り、タグ・見出し・リンクで横断できる個人用の wiki ツリーを生成する CLI ツール。

## 1.3 背景
個人のメモや技術ノートを `.md` で管理していると、量が増えるほど「どこに何を書いたか」が埋もれていく。Obsidian のような高機能エディタは有用だが、特定ツールへロックインされたくない／成果物はプレーンな Markdown で残したいという需要がある。

一方、ソースコードの索引化は `grep` / `ripgrep` / LSP で既に十分に機能しており、専用ツールとして抱える必然性は薄い。

そこで本ツールは対象を **`.md` ファイルのみ** に絞り、メモを投げ込み続けると自動的に wiki が育っていく状態を提供する。

## 1.4 目的
- 書き溜めた md ノートから、再読解可能な wiki を自動構築する
- md を追加しただけで、タグ索引・見出し索引・リンクグラフ・バックリンクが自動更新される "増築型" を実現する
- 成果物は純粋な Markdown のみとし、任意のエディタ・ビューア・CI で扱えるようにする
- AI / LLM を使わないため、API 料金や外部依存なく何度でも再生成できる

## 1.5 生成物の性質
生成物は「自然文の要約」ではなく「構造索引」である。原本の本文は基本的に改変せず、`[[wikilink]]` を標準 Markdown リンクへ変換することだけを行う。索引ページ（タグ／見出し／リンク）は原本を参照する入口として機械的に組み立てる。
